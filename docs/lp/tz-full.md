1. Контекст и цель
ExtraSpace запускает контекстную рекламу в Google Ads. Главная страница сайта (extraspace.kz)
— общая, для рекламного трафика она работает плохо: размытый intent, низкий CR.
Под рекламу нужны 3 узкоспециализированные посадочные — каждая под свой тип услуги и
свою группу запросов. Цель страниц — получить заявку (имя + телефон) в первый же визит, без
долгого изучения сайта.
Главные KPI страниц: - CR клик→заявка ≥ 5% - Все заявки приходят с GCLID (для оптимизации
Google Ads) - Скорость загрузки mobile <2s

3.3 Аналитика и трекинг

3.4 Phone gating (КРИТИЧНО)
На страницах не должно быть прямых ссылок <a href="tel:..."> или открытого номера
телефона в первом экране.
Поведение: 1. На странице есть кнопка «Показать номер» (или «Позвонить») 2. По клику —
открывается модалка с формой: - Поле «Имя» (обязательное) - Поле «Телефон» (обязательное,
маска +7 (___) ___-__-__ ) - Чекбокс согласия с обработкой данных - Кнопка «Отправить» 3.
После успешного submit — модалка показывает номер +7 778 391 1425 + кнопку «Позвонить» (это
tel: ссылка) + кнопку «Написать в WhatsApp» ( wa.me/77783911425 ) 4. Параллельно с показом
номера — заявка отправляется в CRM (см. раздел 3.5)
Почему так: без phone gating звонки приходят без GCLID/UTM-параметров → невозможно понять
с какого ключа пришёл клиент → невозможно оптимизировать рекламу.
3.5 Форма заявки и интеграция
Поля mini-form (phone gating): - name — имя (string, required, min 2 chars) - phone — телефон
(string, required, формат +7XXXXXXXXXX ) - service_type — тип услуги (auto-detected по URL: для
LP-1 = individual , LP-2 = camera , LP-3 = cloud ) - agreement — согласие (boolean, must be
true)
Полный payload, отправляемый при submit (включая трекинг):
Куда отправлять: 1. Основной канал — CRM (та же интеграция что у формы «Обратный звонок»
на главной странице). Если CRM ещё нет — endpoint указать. 2. Дублирование в Telegram —
отдельный bot/chat для уведомлений менеджерам (детали — у клиента) 3. Email уведомление —
на адрес менеджера (опционально)
Защита от ботов: - Honeypot-поле (скрытое поле, заполненное ботами → submit отклоняется) -
Rate limit: 1 успешный submit с одного IP в 60 секунд - reCAPTCHA v3 — НЕ ставить (ломает CR на
мобилке)
3.6 GCLID / UTM passthrough
Логика:
При первой загрузке любой LP: 1. Парсим GET-параметры URL: ?
gclid=...&utm_source=...&utm_medium=...&utm_campaign=...&utm_content=...&utm_term=...
2. Если есть хотя бы один — сохраняем все эти параметры в localStorage под ключом
extraspace_attribution 3. Также сохраняем first_visit_at (timestamp первого визита) и
landing_page (первая LP куда зашли) 4. TTL — 90 дней. На каждом новом визите проверяем —
если запись старше 90 дней, перезаписываем. Если младше — обновляем last_visit_at , но
gclid / utm_* / first_visit_at НЕ меняем (first-touch attribution). 5. Важно: если на новый визит
приходит новый GCLID или UTM — перезаписываем все поля (last-click attribution для повторных
кликов из рекламы).
При submit любой формы — все поля из localStorage добавляются в payload (см. раздел 3.5).

8. Acceptance Criteria — критерии приёмки
Подрядчик сдаёт работу когда выполнено:
Функциональность
Все 3 LP открываются по своим URL
Phone gating работает: клик «Показать номер» → форма → submit → раскрытие номера
Форма отправляет заявку в CRM/Telegram с полным payload включая GCLID и UTM
Тест: открыть /lp/arenda-boksa-almaty?
gclid=test123&utm_source=google&utm_medium=cpc&utm_campaign=test → отправить форму
→ в CRM пришла заявка с этими параметрами
localStorage extraspace_attribution сохраняется при первом визите и подтягивается при
submit на любой странице сайта
Honeypot работает (заполненное скрытое поле → submit отклонён)
Rate limit работает (попытка 2 submit с 1 IP за 60 сек → ошибка)
Аналитика
Все 8 GTM-событий пуши в dataLayer (можно проверить в DevTools console)
GA4 видит все события (тест в GA4 Realtime)
Google Ads conversion tag срабатывает на form_submit_lead
Facebook Pixel и Microsoft Clarity подключены
Производительность
PageSpeed Insights mobile ≥ 85 (тест: pagespeed.web.dev на каждой LP)
Время загрузки <2s на 4G (Chrome DevTools throttling)
LCP <2.5s, CLS <0.1 (Web Vitals)
Mobile / Cross-browser
Тест на iPhone (Safari)
Тест на Android (Chrome)
Тест на десктопе (Chrome / Safari / Firefox)
Без overflow / горизонтального скролла на 320px ширине
SEO
<meta name="robots" content="noindex,nofollow"> на всех 3 LP
LP не попали в sitemap.xml
Schema.org LocalBusiness JSON-LD валиден (тест: search.google.com/test/rich-results)
Open Graph превью красивое в WhatsApp / Telegram (тест: вставить ссылку в чат)
Гео
Карта 2GIS показывает оба филиала
Кнопки «Построить маршрут» работают (открывают приложение 2GIS)
Адреса и телефон корректные на всех 3 LP
